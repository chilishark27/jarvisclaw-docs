# Agent Payments (x402 Protocol)

AI agents can pay for API calls directly from their own wallet using the x402 protocol — no API key required. Payment is made in USDC on Base (EIP-155 chain 8453).

## How it works

1. Agent sends a request without an auth header.
2. Server returns **HTTP 402** with payment requirements (amount, recipient, network, asset).
3. Agent signs an EIP-712 `TransferWithAuthorization` message using its wallet private key.
4. Agent retries the same request with the `PAYMENT-SIGNATURE` header containing a base64-encoded payment proof.
5. Server verifies the signature, executes the on-chain transfer, and returns the API response.

## 402 Response Format

When you hit an endpoint without auth, you'll get back:

```json
{
  "accepts": [
    {
      "payTo": "0x<seller-address>",
      "amount": "50000",
      "network": "eip155:8453",
      "asset": "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
      "maxTimeoutSeconds": 120
    }
  ]
}
```

| Field | Description |
|-------|-------------|
| `payTo` | Recipient wallet address |
| `amount` | Price in smallest unit (USDC has 6 decimals, so `50000` = $0.05) |
| `network` | Chain identifier (`eip155:8453` = Base mainnet) |
| `asset` | Token contract address (USDC on Base) |
| `maxTimeoutSeconds` | How long the signed authorization remains valid |

## Method 1: Raw Python (requests + eth_account)

Full manual implementation of the x402 payment handshake. This gives you complete control over the flow.

```python
import requests
import json
import base64
import os
import time
from eth_account import Account
from eth_account.messages import encode_typed_data

GATEWAY = "https://api.jarvisclaw.com"
PRIVATE_KEY = os.environ["WALLET_PRIVATE_KEY"]  # 0x-prefixed hex

account = Account.from_key(PRIVATE_KEY)

def chat_with_x402(messages: list[dict]) -> dict:
    """Send a chat completion request using x402 payment."""
    url = f"{GATEWAY}/v1/chat/completions"
    payload = {
        "model": "gpt-4o",
        "messages": messages,
    }

    # Step 1: Send request without auth → triggers 402
    resp = requests.post(url, json=payload)

    if resp.status_code != 402:
        # If not 402, might be free or already authed
        return resp.json()

    # Step 2: Parse payment requirements
    payment_info = resp.json()["accepts"][0]
    pay_to = payment_info["payTo"]
    amount = int(payment_info["amount"])
    network = payment_info["network"]
    asset = payment_info["asset"]
    max_timeout = payment_info["maxTimeoutSeconds"]

    # Step 3: Sign EIP-712 TransferWithAuthorization
    now = int(time.time())
    valid_after = now - 60
    valid_before = now + max_timeout
    nonce = os.urandom(32)

    # Extract chainId from network string "eip155:8453"
    chain_id = int(network.split(":")[1])

    typed_data = {
        "types": {
            "EIP712Domain": [
                {"name": "name", "type": "string"},
                {"name": "version", "type": "string"},
                {"name": "chainId", "type": "uint256"},
                {"name": "verifyingContract", "type": "address"},
            ],
            "TransferWithAuthorization": [
                {"name": "from", "type": "address"},
                {"name": "to", "type": "address"},
                {"name": "value", "type": "uint256"},
                {"name": "validAfter", "type": "uint256"},
                {"name": "validBefore", "type": "uint256"},
                {"name": "nonce", "type": "bytes32"},
            ],
        },
        "primaryType": "TransferWithAuthorization",
        "domain": {
            "name": "USD Coin",
            "version": "2",
            "chainId": chain_id,
            "verifyingContract": asset,
        },
        "message": {
            "from": account.address,
            "to": pay_to,
            "value": amount,
            "validAfter": valid_after,
            "validBefore": valid_before,
            "nonce": nonce,
        },
    }

    signable = encode_typed_data(full_message=typed_data)
    signed = account.sign_message(signable)

    # Step 4: Build x402 v2 payload
    x402_payload = {
        "x402Version": 2,
        "resource": {
            "url": url,
            "description": "API request",
            "mimeType": "application/json",
        },
        "accepted": {
            "scheme": "exact",
            "network": network,
            "amount": str(amount),
            "asset": asset,
            "payTo": pay_to,
            "maxTimeoutSeconds": max_timeout,
        },
        "payload": {
            "signature": "0x" + signed.signature.hex(),
            "authorization": {
                "from": account.address,
                "to": pay_to,
                "value": str(amount),
                "validAfter": str(valid_after),
                "validBefore": str(valid_before),
                "nonce": "0x" + nonce.hex(),
            },
        },
        "extensions": {},
    }

    # Step 5: Retry with PAYMENT-SIGNATURE header
    payment_header = base64.b64encode(
        json.dumps(x402_payload).encode()
    ).decode()

    resp = requests.post(
        url,
        json=payload,
        headers={"PAYMENT-SIGNATURE": payment_header},
    )
    return resp.json()


# Usage
result = chat_with_x402([{"role": "user", "content": "Hello!"}])
print(result["choices"][0]["message"]["content"])
```

## Method 2: SDK (automatic x402 handling)

The SDK handles the entire 402 handshake automatically — you just provide your wallet key.

```python
from jarvisclaw import JarvisClaw

# Initialize with wallet private key (x402 mode)
client = JarvisClaw(wallet_private_key="0x_YOUR_PRIVATE_KEY")

# Chat completion — SDK auto-handles 402 payment
response = client.chat.complete("Hello!")
print(response.content)
print(f"Cost: ${response.usage.cost_usd:.6f}")

# Streaming works the same way
for chunk in client.chat.stream("Tell me a joke"):
    print(chunk, end="")

# Image generation (auto-polls until ready)
img = client.image.generate("A futuristic city")
print(img.url)

# Video generation (blocks until ready)
job = client.video.generate("Ocean waves")
print(job.url)

# Text-to-Speech
result = client.audio.speech("Hello world", voice="alloy")

# Web search
results = client.search.query("latest AI news")
```

```shell
pip install jarvisclaw
```

## Comparing the two approaches

| | Raw Python | SDK |
|---|---|---|
| Setup | `pip install requests eth-account` | `pip install jarvisclaw` |
| Auth | Manual 402 → sign → retry loop | Automatic |
| Streaming | Manual SSE parsing | Built-in |
| Async polling | Manual | Built-in |
| Wallet key | You manage signing | Pass once at init |
| Use case | Custom integrations, learning | Production agents |

## Security Notes

- Your private key never leaves the local machine — only the signature is sent.
- Each payment authorization is scoped to a single request (unique nonce, tight validity window).
- The `validBefore` timestamp prevents replay attacks.
- USDC `TransferWithAuthorization` is a gasless meta-transaction — no ETH needed for gas.
